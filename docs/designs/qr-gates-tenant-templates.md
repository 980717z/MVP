# CEO Plan: QR 合约 Gates + 租户模版内核 (Tenant Template Core)
Generated by /plan-ceo-review on 2026-07-05
Branch: main | Mode: SELECTIVE EXPANSION

## Problem

1. **不变量裸奔**：印在物理牌子上的 QR 合约（tenant slug、桌号 labels、`/menu/[tenant]?t=&m=` URL 形状）目前只受 CLAUDE.md 文字保护。并行 AI session 已两次默默回退过功能（git `774d356`）。
2. **入驻无底层**：fulai 是手工 SQL 一天堆出来的。需要一套底层逻辑：模版 + 内容填充，短期服务运营者（Allen + Claude Code），长期支撑向导 UI。

## Key Reframes（评审共识）

- BentoOS 卖 personalized OS（white-glove 是卖点）→ self-serve 短期是**运营者提效工具**，不是商家功能。
- **威胁模型是"拿着 owner 权限的 AI session"**，不只是人类手滑（外部意见 OV3）→ 解锁必须 SQL-editor-only。
- 向导 UI 服务的用户现在不存在（OV7）→ 本次砍到"店型选择"一步，完整向导已入 TODO（触发条件：商家 #2）。

## Architecture: ONE schema, TWO frontends（向导前端延后）

```
                ┌────────────────────────────────┐
                │   TenantTemplate JSON (v1)      │  templates/*.json
                │  version/type/name/modules/     │  ← chinese-restaurant.json
                │  tables/categories/delivery/    │    (E1 倒导, 自动匿名化)
                │  hours/menu(全量: variants+     │  ← counter-service.json
                │  market+image, template_key)    │    (E5, 无桌码/纯自取/重会员)
                └──────────┬─────────────────────┘
              validate() + provision plan (lib/tenantTemplate.ts)
                     ┌─────┴──────────────┐
        本次: CLI → SQL 生成              延后(TODO): 完整向导
        scripts/provision-sql.ts          本次仅: 建店页加
        lit() 转义 + 单测                  「选店型→预选模块」一步
        生成的 SQL 即预览 (E6)
```

## Gates（四层，含外部意见修正）

```
  层        │ 防什么                    │ 机制
  ──────────┼──────────────────────────┼────────────────────────────────
  DB 触发器 │ owner 权限的 AI/员工      │ qr_locked_at 非空时: slug 不可改;
            │ 的破坏性写入              │ tables 只增 (NEW @> OLD);
            │                          │ **DELETE 禁止 (OV1)**;
            │                          │ **解锁仅限 SQL 编辑器
            │                          │  (auth.uid() IS NULL, OV3)**
  UI 锁     │ 手滑、误操作              │ 锁定徽章 + 锁定按钮
            │                          │ (无 UI 解锁 — OV3 砍掉 E4)
  部署门禁  │ 代码层重构改坏 URL 形状   │ lib/qrContract.ts 常量 + vitest
            │                          │ 守卫; **build = vitest run &&
            │                          │ next build (OV2, 测试红=部署失败)**
  审计日志  │ gate 拦不住的合法修改     │ tenants_audit: SECURITY DEFINER
            │                          │ 触发器 + RLS 全拒 (OV6)
  体检      │ "gate 根本没装上"的静默   │ supabase/health-check.sql 查
            │ 失败 (手动迁移漂移, OV8)  │ pg_trigger/pg_constraint/锁状态
```

## Accepted Scope (final deliverables)

1. `supabase/qr-lock.sql` — qr_locked_at + protect_qr_contract 触发器（UPDATE+DELETE、slug 锁、tables 只增、SQL-only 解锁、QR_LOCKED_* 错误码）+ tenants_audit（SECURITY DEFINER + RLS 全拒）+ slug CHECK/保留字 + menu_items.template_key 部分唯一索引 + 锁定 fulai
2. `supabase/health-check.sql` — 可重复体检
3. `lib/qrContract.ts` + 守卫测试；package.json build 改 `vitest run && next build`
4. QrMenuPortal 锁定徽章 + 锁定按钮 + 错误码三语映射（无解锁 UI）
5. `lib/tenantTemplate.ts` — schema v1（全量菜单形状）+ validate + 幂等 provision 计划
6. `scripts/export-template.ts`（自动匿名化）→ `templates/chinese-restaurant.json`；手写 `templates/counter-service.json`
7. `scripts/provision-sql.ts` — lit() 转义 + 单测；生成 SQL 即预览
8. 建店页加「选店型→预选模块」一步（仅此，完整向导延后）
